How It Works

Three steps. That's it.

01

Build your mockup

Create your design as HTML + CSS. Use any framework or plain markup â€” if a browser can render it, Design Reviewer can display it.

02

Push it

$ design-reviewer push ./mockup --name "Homepage v2"

Your files are uploaded and a new version is created automatically.

03

Review together

Open the browser, click on the design to drop comments. Teammates see feedback pinned to the exact spot. Resolve threads when done.

CLI Reference

Your terminal is the interface.

$ design-reviewer login --server URL

Authenticate via Google OAuth

$ design-reviewer push <dir> --name "Name"

Upload a design directory as a new version

$ design-reviewer init

Generate a DESIGN_GUIDELINES.md template

$ design-reviewer logout

Remove stored credentials

Get Started

Up and running in 2 minutes.

# Clone and run $ git clone https://github.com/ab/design-reviewer.git $ cd design-reviewer $ cp .env.template .env # add your Google OAuth creds $ go run ./cmd/server # Build the CLI and push your first design $ go build -o design-reviewer ./cmd/cli $ ./design-reviewer login --server http://localhost:8080 $ ./design-reviewer push ./my-mockup --name "First Design"
